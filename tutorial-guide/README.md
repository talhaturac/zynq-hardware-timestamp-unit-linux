# Tutorial-Guide

## 1. Hardware Design (Vivado)
Open the Vivado project located in vivado-project/hw-design.

The Block Design includes the Zynq Processing System and the custom 64-bit TSU IP core.

The IP core registers are mapped to the AXI interconnect.

Generate the bitstream and export the hardware (.xsa file) to be used by PetaLinux.

## 2. PetaLinux & Kernel Configuration

### Device Tree Configuration

The custom driver requires a specific compatible string and interrupt configuration. This is defined in system-user.dtsi:

```DTS
/* project-spec/meta-user/recipes-bsp/device-tree/files/system-user.dtsi */
/include/ "system-conf.dtsi"

&axi_tsu_64bit_0 {
    compatible = "talha,ttt-tsu-driver";
    interrupt-parent = <&intc>;
    interrupts = <0 29 4>; /* SPI, IRQ 29, Active-High Level-Sensitive */
};
```

### Kernel Configuration (Ftrace)

To analyze system latency and driver execution times, Ftrace is enabled in the kernel.
Run petalinux-config -c kernel and enable the following under Kernel hacking ---> Tracers:

```
[*] Kernel Function Tracer

[*] Kernel Function Graph Tracer

[*] Interrupts-off Latency Tracer

[*] Scheduling Latency Tracer

[*] Trace process context switches and events
```

Note: Enabling these tracers might trigger a trace_printk() debug warning during boot. This is expected and safe for debugging purposes.

## 3. Ubuntu RootFS Setup

⚠️ Note on Ubuntu RootFS: Due to GitHub's 100 MB file size limit, the full uncompressed Ubuntu 22.04 LTS (armhf) RootFS is not included in this repository.

To recreate the exact RootFS used in this project, we utilize debootstrap and QEMU.

Rebuilding the RootFS via Debootstrap
On your host machine, run the following commands to create the rootfs:

```Bash
mkdir ubuntu-rootfs && cd ubuntu-rootfs
sudo apt update && sudo apt install qemu-user-static debootstrap

# Download Ubuntu 22.04 (Jammy) for 32-bit ARM (armhf)
sudo debootstrap --arch=armhf --foreign jammy ./rootfs [http://ports.ubuntu.com/ubuntu-ports/](http://ports.ubuntu.com/ubuntu-ports/)

# Copy QEMU static binary to execute ARM instructions on host PC
sudo cp /usr/bin/qemu-arm-static ./rootfs/usr/bin/

# Mount necessary systems
sudo mount -t proc /proc ./rootfs/proc
sudo mount -t sysfs /sys ./rootfs/sys
sudo mount -o bind /dev ./rootfs/dev

# Chroot into the ARM environment
sudo chroot ./rootfs /bin/bash

# Inside chroot: Complete stage 2
/debootstrap/debootstrap --second-stage
apt update && apt install sudo nano net-tools openssh-server build-essential kmod -y

# Add user and set host
adduser talha
usermod -aG sudo talha
echo "zybo-tsu" > /etc/hostname
exit

# Unmount
sudo umount ./rootfs/proc ./rootfs/sys ./rootfs/dev
```

## 4. Booting the System

You can boot the board via TFTP by interrupting U-Boot over the serial console:

```
Zynq> setenv serverip 192.168.1.100
Zynq> setenv ipaddr 192.168.1.10
Zynq> tftpboot 0x10000000 image.ub
Zynq> bootm 0x10000000
```

## 5. Running the Application & Ftrace Verification

Once logged into the Ubuntu environment on the Zybo-Z7, you can test the driver and hardware interaction.

### Loading the Driver

```Bash
sudo insmod /home/talha/ttt-tsu-driver.ko
lsmod | grep ttt_tsu_driver
```

### Running the Userspace Test

Compile and run the test application (tsu_test.c). This application opens /dev/ttt_tsu_device, starts the counter, reads it 5 times, and triggers a Soft-IRQ to test ftrace integration.

```Bash
gcc -O3 tsu_test.c -o tsu_test
sudo ./tsu_test

#Expected Output:

Starting hardware counter...
Timestamp [0]: 125060837
Timestamp [1]: 145061120
...
Triggering Soft-IRQ for ftrace log...
Stopping hardware counter...
Test completed successfully.
```

### Ftrace Verification

To verify the Soft-IRQ execution and inspect the latency logs generated by trace_printk(), check the kernel debug tracing directory:

```Bash
sudo cat /sys/kernel/tracing/trace | grep ttt_tsu
```

This will display the exact timestamp and context switch logs generated during the test execution, confirming the deterministic behavior of the TSU and the interrupt handler.
